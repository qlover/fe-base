# Next Seed (`examples/next-seed`)

> 中文: [README.md](./README.md)

**Template purpose**: A slim Next.js 16 app shell (App Router primary, plus a Pages Router `/admin` example) with **theming**, **i18n**, and **login / register / session** out of the box.

> This is **not** an OAuth 2.0 authorization server example (no `/oauth/authorize`, `/oauth/token`, developer console, or client management). For an OAuth AS example, see `examples/next-oauth`.

## What's included

- **Theming**: light / dark switching (`ThemeSwitcher`), built on `@wrksz/themes`, `@qlover/tailwind-theme`, and Tailwind CSS
- **i18n**: `next-intl` with an in-house `i18n-identifier` / `i18n-mapping` convention, `en` / `zh` locale-prefixed routes
- **Auth**: Supabase Auth-backed email/password login and registration, email/phone OTP, third-party sign-in (GitHub / Google SSO), and HttpOnly JWT session cookies
- **Simple admin dashboard**: a `/admin` template page (Pages Router), gated behind login

## What's not included

- OAuth authorization server / clients / playground
- `@qlover/oauth-wrapper`
- Request logs, AI, Swagger, and other extras (see `next-oauth` for the full example)

## Quick Start

From the monorepo root:

```bash
pnpm --filter examples/next-seed install
cp examples/next-seed/.env.template examples/next-seed/.env.local
# Fill Supabase and SESSION_* vars per the comments
```

**Required env vars** (full list in `.env.template`):

| Variable                                   | Purpose                                                              |
| ------------------------------------------ | -------------------------------------------------------------------- |
| `SITE_URL` / `NEXT_PUBLIC_SITE_URL`        | Public site URL                                                      |
| `SUPABASE_URL` / `SUPABASE_ANON_KEY`       | Supabase project connection                                          |
| `SESSION_SECRET`                           | Signs the HttpOnly login session cookie                              |
| `SESSION_KEY` (alias: `OAUTH_SESSION_KEY`) | Session cookie name                                                  |
| `NEXT_PUBLIC_STRING_ENCRYPT_KEY`           | Shared key for encrypting the password field in login/register forms |

Optionally run `makes/sql/001-base-tables.sql` in the Supabase SQL editor, then:

```bash
pnpm --filter examples/next-seed dev          # http://localhost:3010
pnpm --filter examples/next-seed start        # after build, default 3011
```

Common scripts: `type-check`, `test` (Vitest), `lint`, `format` — see `package.json`.

Default ports: **3010** (dev), **3011** (start).

## Project Layout

```
server/   # Backend: controllers, services, providers (SupabaseAuthProvider), repositorys, IOC
src/      # Frontend: app/ (App Router pages + API), pages/ (Pages Router, e.g. /admin), uikit/, impls/
shared/   # Shared across server/client: config (routes, i18n), interfaces, schemas, validators
```

- **Route constants**: `shared/config/route.ts`, `shared/config/apiRoutes.ts` (the latter is auto-generated by `tools/generateApiRoutes.ts` on `next dev`/`build` — do not edit by hand)
- **i18n convention**: `shared/config/i18n-identifier/` (copy keys with `@localZh`/`@localEn` doc comments) → `shared/config/i18n-mapping/` (the subset of keys a page/component needs). See [docs/i18n.en.md](./docs/i18n.en.md)
- **Auth implementation**: `server/providers/SupabaseAuthProvider.ts`, `server/services/SessionService.ts`, `server/services/AuthUserService.ts`
